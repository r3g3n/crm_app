# Документация API для CRM системы

API предоставляет доступ к основным сущностям CRM: контактам (клиентам), их статусам, комментариям и истории. Все запросы к API (кроме авторизации) требуют наличия токена.

## Базовый URL
Все запросы должны выполняться к: `/api/v1`

## Авторизация
Аутентификация в API осуществляется на основе токенов (Bearer Token).

### Получение токена
- **URL**: `/auth/login`
- **Метод**: `POST`
- **Тело запроса** (JSON):
  ```json
  {
    "username": "ваше_имя_пользователя",
    "password": "ваш_пароль"
  }
  ```
- **Успешный ответ** (200 OK):
  ```json
  {
    "success": true,
    "token": "ey...",
    "user": {
      "id": 1,
      "username": "admin",
      "role": "admin"
    }
  }
  ```

### Использование токена
Во все последующие запросы необходимо добавлять HTTP заголовок:
`Authorization: Bearer ВАШ_ТОКЕН`

## Контакты (Клиенты)

### 1. Получение списка контактов с фильтрацией
- **URL**: `/contacts`
- **Метод**: `GET`
- **Параметры запроса** (Query parameters):
  - `page` (int) — Номер страницы (по умолчанию 1)
  - `per_page` (int) — Количество элементов на странице (по умолчанию 50, максимум 100)
  - `q` (string) — Строка для поиска (по всем текстовым полям)
  - `status` (string) — Фильтр по статусу (например, `Новый`, `В работе`)
  - `sort` (string) — Поле сортировки: `id` (по умолчанию) или `updated_at` (сначала новые)
- **Успешный ответ** (200 OK):
  ```json
  {
    "success": true,
    "data": [
      {
        "id": 1,
        "email": "example@mail.com",
        "status": "Новый",
        "comment_count": 2,
        ... // остальные поля
      }
    ],
    "meta": {
      "total": 150,
      "page": 1,
      "pages": 3,
      "per_page": 50
    }
  }
  ```

### 2. Получение детальной информации о контакте (включая отчет и письмо)
- **URL**: `/contacts/<id>`
- **Метод**: `GET`
- **Успешный ответ** (200 OK):
  ```json
  {
    "success": true,
    "data": {
      "id": 1,
      "email": "example@mail.com",
      "status": "Новый",
      "detailed_report": "<p>Подробный отчет...</p>",
      "first_mail": "<p>Текст письма...</p>",
      "comments": [ ... ],
      "history": [ ... ],
      ... // остальные поля
    }
  }
  ```

### 3. Добавление нового контакта
- **URL**: `/contacts`
- **Метод**: `POST`
- **Тело запроса** (JSON, все поля опциональны):
  ```json
  {
    "email": "new_client@mail.com",
    "phone": "+79991234567",
    "site": "example.com",
    "description": "Описание клиента",
    "status": "Новый",
    "category": "Импортер"
  }
  ```
- **Успешный ответ** (201 Created):
  ```json
  {
    "success": true,
    "contact_id": 123
  }
  ```

### 4. Редактирование контакта (в т.ч. отчета и письма)
- **URL**: `/contacts/<id>`
- **Метод**: `PUT`
- **Тело запроса** (JSON). Можно передавать только те поля, которые нужно обновить:
  ```json
  {
    "detailed_report": "<p>Обновленный отчет по компании</p>",
    "first_mail": "<p>Новый драфт письма</p>",
    "phone": "+70000000000"
  }
  ```
- **Успешный ответ** (200 OK):
  ```json
  {
    "success": true
  }
  ```

### 5. Удаление контакта
- **URL**: `/contacts/<id>`
- **Метод**: `DELETE`
- **Описание**: Удаляет контакт, а также все связанные с ним комментарии и историю изменений.
- **Успешный ответ** (200 OK):
  ```json
  {
    "success": true
  }
  ```

## Статусы и Комментарии

### 1. Обновление статуса контакта
- **URL**: `/contacts/<id>/status`
- **Метод**: `PUT` (или `POST`)
- **Тело запроса** (JSON):
  ```json
  {
    "status": "В работе"
  }
  ```
- **Успешный ответ** (200 OK):
  ```json
  {
    "success": true,
    "status": "В работе",
    "color": "primary"
  }
  ```

### 2. Получение списка комментариев
- **URL**: `/contacts/<id>/comments`
- **Метод**: `GET`
- **Успешный ответ** (200 OK):
  ```json
  {
    "success": true,
    "data": [
      {
        "id": 10,
        "content": "Созвонились, запросили прайс",
        "created_at": "2023-10-15T12:00:00.000",
        "username": "admin",
        "avatar_url": "/static/avatar.png"
      }
    ]
  }
  ```

### 3. Добавление комментария
- **URL**: `/contacts/<id>/comments`
- **Метод**: `POST`
- **Тело запроса** (JSON):
  ```json
  {
    "content": "Клиент попросил перезвонить завтра."
  }
  ```
- **Успешный ответ** (201 Created):
  ```json
  {
    "success": true,
    "comment": {
      "id": 11,
      "content": "Клиент попросил перезвонить завтра.",
      "created_at": "2023-10-15T12:05:00.000",
      "username": "api",
      "avatar_url": "/static/avatar.png"
    }
  }
  ```

### 4. Удаление комментария
- **URL**: `/comments/<id>`
- **Метод**: `DELETE`
- **Описание**: Удаляет комментарий по его ID. Доступно для авторизованных пользователей.
- **Успешный ответ** (200 OK):
  ```json
  {
    "success": true
  }
  ```

## Коды ошибок
- `400 Bad Request` — неверные параметры запроса (например, неверный статус или пустое тело комментария).
- `401 Unauthorized` — токен отсутствует, просрочен или недействителен.
- `404 Not Found` — запрашиваемый контакт не найден в базе.
